Text-to-Sign API Documentation

  • Last updated: 11 April 2023
  • by SignMate Team

Implementation of a text-to-sign API that uses natural language processing (NLP) techniques to tokenize, lemmatize, and remove stopwords from input text. The API also identifies the tense of the input text and adds a specific word to specify tense in the output sign language animation.

The API then uses the resulting words to search for corresponding sign language animation files in a database. If an animation is found, its file path is added to a list of animation file paths that are returned as the output of the API. If an animation is not found, the API splits the word into individual characters and searches for animations corresponding to each character.

It's important to note that this code snippet alone may not be sufficient as a complete API documentation. Additional information such as API endpoints, input and output formats, and authentication mechanisms would be required to provide comprehensive API documentation.

Endpoint

The endpoint for this API is /text-to-sign.

Request Method

The API accepts POST requests.

Request Parameters

The API expects a JSON payload with the following fields:

  • sen : The sentence to be converted to sign language animation.

Response

The API returns a JSON response with the following fields:

  • words : An array of URLs to the sign language animations for each word in the input sentence.
  • text : The input sentence.
